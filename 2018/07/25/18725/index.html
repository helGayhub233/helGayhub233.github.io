HTML 即超文本标记语言，它不是一种编程语言而是一种标记型语言(markup language)。“超文本”是指页面内可以包含图片、链接，甚至音乐、程序等. . .非文字元素，“标记语言”是标准通用标记语言下的一个应用，也是一种规范，一种标准，它通过标记符号来标记要显示的网页中的各个部分。

那么，一张好看的网页是如何做成的？

简单的说，HTML用来设计网页整体框架布局，CSS用来给网页加上丰富的色彩，而Javascript让网页看起来更生动、更灵活。

  • Client / Server (客户端/服务器)简称: C/S网络架构

标签格式

HTML标记大致分为两类：

  • 双边标记: 是指有开始和结束标记，内容放在开始和结束标记之间。

    语法格式：<标签 属性1 = “值1” 属性2 = “值2”>内容</标签>

  • 单边标记: 是指只有开始标记，没有结束标记，一般起一个特殊的功能。

    语法格式：<标签 属性1 = “值1” 属性2 = “值2”/>


编写规范

编写规范的注意事项：

  • HTML标记不区分大小写。如：font、Font、FOnt
  • HTML标记属性可有可无。有的标记是没有属性的，如: html、head、title等
  • HTML的双边标记的内容在开始和结束标签之间，单边标记没有内容
  • HTML标记可以相互嵌套，但注意一定是顺序嵌套


<head>标记介绍

meta标记：位于网页的头部(head)中，包含属性但不包含任何内容。

用于提供有关页面的元信息，如针对搜索引擎和更新频度的描述和关键词

  • http-equiv：文件头原信息，主要目的是服务器向客户端返回时，用什么格式显示
    • refresh：每隔多少秒刷新网页
  • name：主要用于设置网页的搜索关键字、版权信息、作者等
    • keywords：网页搜索关键字
    • description：网页描述信息
  • text/html：字符集，当网页乱码时需要声明相关编码才可正常显示
  • content-type：内容类型

XHTML

传统的HTML的开发初衷是面向PC端，而随着移动终端的不断出现，HTML已经满足不了市场的需求，为此取代HTML的XHTML诞生。

XHTML是一种可扩展超文本的标注语言，标记与HTML一模一样，但是语法要比上一代严格的多。


编写规范

  • 所有的标记和属性都要小写
  • 单边标记必须关闭
  • 所有的属性值必须要加引号
  • 所有的属性必须有值，如：<noshade = noshade>
  • 标记之间要顺序嵌套，外层套内层，一层套一层
  • XHTML 网页必须要有DTD文档类型定义代码


DTD文档类型

DTD文档类型声明要在网页的最顶部，一共有三大类：

  • Strict 严格型：不能再使用各种表现标记：如：font、b、bgcolor（使用CSS取代）
  • Transitional 过度型：继承HTML中的表现写法，继续使用
  • Frameset 框架型：用框架来划分区域

表单的概念

表单主要用来获取客户端用户数据（信息）的。比如：注册表单、查询表单、登录表单等..
它的工作原理，是当浏览带有表单的网页填写了一些必要的信息，然后点击某个按钮进行提交。
这些表单数据，通过互联网传递到了服务器上，服务器中有专门的程序，对表单数据进行认证。如果验证成功，将你的数据存入数据库（MySQL等）,否则返回一个验证成功的信息，如果验证失败，将返回一个错误的信息。

GET 方法和 POST 方法

GET提交方式：
GET 方式，是将表单数据追加到action指定的处理程序后面，然后向服务器发出请求。浏览器传递数据默认为GET方式。
说明：

  • login.php：是表单处理程序文件
  • username=admin&userpwd=123456：表单提交的数据，又称为”查询字符串”
  • action：文件和查询字符串之间用“？”分隔
    • 如果表单action为空，那么表单数据提交给自己处理
    • 每两个表单元素的”名称=值”之间用”&”分隔
    • 表单名称和表单值之间用”=”分隔

注意：如果某个表单元素，不想往服务器传递数据，那么可以不给它加name属性。

特点：

  • GET 方式不能提交敏感数据，如：密码
  • GET 方式只提交少量数据，因为地址栏的长度有限制，大约100外的字符
  • GET 方式下不能上传附件

POST提交方式：
POST方式，它不是将表单数据追加到地址上，而是直接传给表单处理程序。

  • POST 提交的数据相对安全
  • POST 可以提交海量数据
  • POST 方式可以上传附件

普通框架

框架的概念（框架技术）：将一个浏览器窗口划分成若干个小窗口，每一个小窗口显示一个独立的网页。比如：一个窗户由窗格(框架集)和玻璃(框架页)构成，那么框架集用来规划窗格，框架页用来确定每个窗格中放的玻璃，即先有结构，后有内容。

框架集和框架页

框架网页的DTD类型必须是:frameset.dtd

框架集：主要用来划分窗口
框架页：主要用来指定窗口默认显示的网页地址

效果：


<frameset>属性

注意：cols属性和rows属性，每个框架只能选其中一个不能同时使用。

  • cols：划分左右型框架
    • cols=”200,*” //左框架的宽度为200px,剩下的都是右框架比例
    • cols=”180,*,180” //左右框架的宽分别为180px,剩下都是中间框架比例
    • cols=”20%,*” //划分框架时，可以用百分比来表示
  • rows：划分上下型框架
    • rows=”200,*” //上框架的高度为200px，剩下都是下框架比例
    • rows=”180,*,180” //上框架和下框架高分别为180px，剩下都是中间框架
  • frameborder：是否显示框架的边框线，取值：1 或 0
  • border：边框线的粗细
  • bordercolor：边框颜色

<frame>框架页的属性

  • src：当前小窗口中，默认显示的网页地址
  • noresize：不能调整小窗口的大小，如：<noresize = “norsesize”>
  • scolling：是否显示滚动条，取值：auto、yes、no
  • name：给当前小窗口起个名字，这个name就是给a标记target属性来用的
